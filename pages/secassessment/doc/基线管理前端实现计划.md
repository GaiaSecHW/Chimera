> 模块路径:`pages/secassessment/sec-baseline-mgmt/`

## 1. 背景与目标

在 Chimera 平台前端实现"安全功能基线管理"模块,对接已实现的后端 REST API:

- 开发期前端**直连**后端服务(`http://localhost:8000`)
- 预留后期在 **k8s 环境**经 ingress/gateway 访问后端服务的接口(通过环境变量切换,不改代码)
- UI 风格遵循 Chimera 平台设计系统(CSS 变量驱动 + design-system 组件)
- 全部改动限定在 `pages/secassessment/` 目录下,不触碰 `app/`、`types/`、`clients/`、`vite.config.ts`、`layout/` 及任何既有模块

## 2. 调研结论

### 2.1 后端(FastAPI :8000)

- **CORS**:`CORS_ORIGINS` 已含 `http://localhost:3000`,dev 期前端直连后端(不经 Chimera 网关)
- **鉴权**:`CURRENT_USER_PROVIDER=mock`(dev,固定开发用户,忽略 token);`chimera` provider 为生产留桩(将从 chimera token 解析 `{person_id, person_name, role}`)。因此前端复用 `getHeaders()` 携带 chimera_token 即可:dev 被 mock 忽略,prod 由 chimera provider 校验
- **同步**:`SYNC_CLIENT=mock`(dev 模拟延迟+成功率);`sep`(生产接 sep_system,TBD)
- **路由域**(`/api` 前缀):
  - `GET /api/product-org/tree` — 组织树(BG→BU→Product,含 children 嵌套)
  - `POST /api/product-org`、`PUT/DELETE /api/product-org/{node_id}` — 组织节点 CRUD
  - `GET /api/baselines?product_org_id=&keyword=&sync_status=` — 基线列表(**支持组织子树过滤** + 关键字 + 同步状态);返回 `BaselineWithProduct[]`(含 bg_name/bu_name 面包屑)
  - `GET /api/baselines/{id}` — 详情(`BaselineDetail`,含 stats)
  - `PUT/DELETE /api/baselines/{id}` — 编辑基本信息 / 删除(级联)
  - `POST /api/baselines/preview` — multipart xlsx 解析预览(不落库),返回 `BaselinePreview`(stats + nodes)
  - `POST /api/baselines` — multipart 创建(file + 表单字段,表单权威 xlsx 回退),返回 `BaselineDetail`
  - `GET /api/baselines/import-template` — 下载 xlsx 模板(FileResponse)
  - `GET /api/baselines/{id}/nodes` — 节点扁平列表(`NodeOut[]`,含 parent_id,前端建树)
  - `POST /api/baselines/{id}/nodes`、`PUT/DELETE /api/baselines/{id}/nodes/{node_id}` — 节点 CRUD(级联删)
  - `POST /api/baselines/{id}/sync` — 触发同步,返回 `SyncResult`
  - `GET /api/baselines/{id}/logs` — 操作日志(`LogOut[]`)
  - `GET /api/baselines/{id}/events` — 事件(`EventOut[]`)
  - `GET /health`、`GET /api/me`
- **数据模型要点**:
  - `sync_status`: `unsync` / `syncing` / `synced` / `sync_failed`
  - `node_type`: `level1` / `level2` / `item`
  - `priority`: `high` / `medium` / `low`
  - `sources`:level2 = 字符串数组;item = `[{document, section}]`
  - `BaselineStats`:total_items / mapped_items / unmapped_items / total_level1_dimensions / total_level2_dimensions / level2_dimensions_with_items / level2_dimensions_empty / mapping_coverage_percent

### 2.2 前端模式(Chimera)

- **HTTP 基础**(`clients/base.ts`):`API_BASE=''`、`getHeaders()`(JSON + `Bearer <chimera_token>`)、`getAuthHeaders()`(仅 bearer,用于 blob/multipart)、`handleResponse()`(401 清 token 并派发 `chimera-unauthorized`、非 2xx 抽取 detail/error/message 拼接并抛出带 `.status` 的 Error)
- **设计系统**(`design-system/`):`PageHeader`、`PageSection`、`StatisticCard`、`Toolbar`/`SearchInput`、`DataTable`(+ `Pagination`,列定义含 `render/sortable/sortKey/align/width`)、`Modal`(portal + 焦点陷阱 + ESC)、`FormField`、`FormActionBar`、`EmptyState`、`Button`、`Input`、`Select`、`DropdownSelect`、`SegmentedControl`、`Card`。`StatusBadge` 在 `components/StatusBadge.tsx`(非 design-system)
- **反馈**:`showConfirm`/`showAlert`/`showPrompt`(`components/DialogService.tsx`,全局 promise-based,无需挂载节点);`useUiFeedback()`(本地 hook,需渲染 `feedbackNodes`)
- **CRUD 参考**:`pages/platform/RoleMgmtPage.tsx`(简单 list+create+edit+delete)、`pages/project/ProjectMgmtPage.tsx`(服务端分页 + `DataTable pagination`)
- **自治子注册表**:`pages/secassessment/viewRegistry.tsx` 已建立,框架(`app/viewRegistry.tsx`)用 `sec-assessment-` / `sec-baseline-` 双前缀守卫并委托;新增 view 仅在此文件加 case/startsWith 分支,**永不再改 `app/` 与 `types/`**
- **路由**:`HashRouter` + `:view` 通配;`setCurrentView` 驱动导航
- **现有占位页**:`pages/secassessment/sec-baseline-mgmt/SecBaselineMgmtPage.tsx`(开发中占位,本次替换)

## 3. API 接入方案(直连 + k8s 预留)

`client.ts` 顶部:

```ts
import { getHeaders, getAuthHeaders, handleResponse } from '../../../clients/base';

// 开发期直连后端;k8s 环境由运维设 VITE_SEC_BASELINE_API_BASE='' (空) 切换为相对路径,
// 经 ingress/gateway 路由到基线服务,前端代码无需改动。
const SEC_BASELINE_API_BASE =
  String(import.meta.env.VITE_SEC_BASELINE_API_BASE ?? 'http://localhost:8000');
const PREFIX = `${SEC_BASELINE_API_BASE}/api`;
```

- **Dev**:直连 `http://localhost:8000`,靠后端 CORS(已含 localhost:3000);`getHeaders()` 携 chimera_token(mock 忽略)
- **K8s 预留**:运维设 `VITE_SEC_BASELINE_API_BASE=`(空)→ 请求变相对路径 `/api/...`,由 ingress 路由;或设为网关前缀(如 `/api/sec-baseline-mng`,需 ingress 同步 rewrite)
- **multipart / blob**:用 `getAuthHeaders()`(仅 bearer,不设 Content-Type,让浏览器自动加 boundary)
- **不改 `vite.config.ts`、不改 `clients/api.ts`**:client 自带 base,types 自带,完全自包含

## 5. view 命名(均 `sec-baseline-` 前缀,已被框架守卫)

**Sidebar 嵌套结构**:`sec-assessment` 顶级 Tab 下,"安全功能基线管理"为可展开父项(`id='sec-baseline'`,`defaultExpanded:true`),含 2 子项:`基线列表`(sec-baseline-mgmt)、`组织树管理`(sec-baseline-org-tree)。父项点击仅展开/折叠,子项点击导航;进入 Tab 时父项默认展开。`sec-baseline-create` 与 `sec-baseline-detail-{id}` 不进 sidebar(由列表页按钮/行进入)。

| view | 渲染组件 | 备注 |
|---|---|---|
| `sec-baseline` | `SecBaselineMgmtPage` | 父项分组键(兜底,直访时渲染列表页) |
| `sec-baseline-mgmt` | `SecBaselineMgmtPage` | 列表页(sidebar 子项;已注册,本次替换实现) |
| `sec-baseline-create` | `SecBaselineCreatePage` | 创建向导(case,列表页"新增基线"进入) |
| `sec-baseline-detail-{id}` | `SecBaselineDetailPage` | 详情页(`startsWith` 解析 id,参考 secocto detail 模式) |
| `sec-baseline-org-tree` | `SecBaselineOrgTreePage` | 组织树管理页(sidebar 子项,case) |

`viewRegistry.tsx` 改动示意:

```tsx
switch (ctx.currentView) {
  case 'sec-baseline':
  case 'sec-baseline-mgmt':
    return <SecBaselineMgmtPage onNavigateToView={ctx.setCurrentView} />;
  case 'sec-baseline-create':
    return <SecBaselineCreatePage onNavigateToView={ctx.setCurrentView} />;
  case 'sec-baseline-org-tree':
    return <SecBaselineOrgTreePage onNavigateToView={ctx.setCurrentView} />;
}
if (ctx.currentView.startsWith('sec-baseline-detail-')) {
  const id = ctx.currentView.replace('sec-baseline-detail-', '');
  return <SecBaselineDetailPage baselineId={id} onNavigateToView={ctx.setCurrentView} />;
}
return null;
```

## 6. 页面设计(参考原型,非照搬)

### 6.1 列表页 `SecBaselineMgmtPage`(view `sec-baseline-mgmt`)

- 布局:`flex` 横向 — 左 `OrgTreePanel`(w-60,只读组织树,过滤)+ 右内容区
- 右内容区:`PageHeader`(标题"基线列表" + 刷新 + "新增基线"→`sec-baseline-create`)→ 4 `StatisticCard`(基线总数 / 待同步 / 同步失败 / 平均覆盖率,按当前过滤集合计算)→ Toolbar(关键字搜索 + 同步状态筛选 + 组织面包屑 + 结果计数)→ `DataTable`
- 表格列:基线(名+code+uuid)/ 所属产品(名+面包屑)/ 分类 / 版本 / 检查项(总数+已映射/未映射)/ 覆盖率(进度条+百分比)/ 同步状态(badge+上次同步)/ 创建人 / 创建时间 / 操作
- 行操作:查看(→`sec-baseline-detail-{id}`)、同步(`showConfirm` → `POST /sync`)、编辑(→`sec-baseline-detail-{id}` 直达 edit)
- 数据:`listBaselines({ product_org_id: selectedOrgId, keyword, sync_status })`,选中组织节点变化即重取
- "管理组织树"入口 → `sec-baseline-org-tree`

### 6.2 详情页 `SecBaselineDetailPage`(view `sec-baseline-detail-{id}`)

- `PageHeader`:返回(→`sec-baseline-mgmt`)+ 标题 + 副标题(code · 版本 · 产品路径 · 创建人 · 同步 badge)+ "更多"菜单(编辑基本信息 / 删除)+ "手动同步"按钮
- 5 Tab(`SegmentedControl` 或 tab 按钮):
  1. **概览**:基本信息 grid(名称/英文/编码/分类/版本/产品/创建人/时间)+ 检查项统计 tiles(8 项 stats)
  2. **节点树**:左树(w-72,虚拟根=基线)+ 右列表/详情;非叶子选中显示叶子列表(可筛选 keyword/priority/is_key_ability,行内展开只读详情);叶子选中显示详情卡;`NodeEditorModal` 支持新增(level1/level2/item 按父类型)/编辑/查看;删除级联(`showConfirm` 提示子节点数)
  3. **同步**:3 指标卡(当前状态/同步次数/最后时间)+ 同步事件流 timeline(过滤 event_type 含 sync)
  4. **操作日志**:`DataTable`(时间/操作 badge/对象/描述/操作人)
  5. **事件**:`DataTable`(时间/事件类型/对象/状态变更/描述/触发人)
- 数据:`getBaseline(id)` + `listNodes(id)`(建树)+ `getLogs(id)` + `getEvents(id)`,按 Tab 懒加载
- 编辑基本信息:`Modal` + `FormActionBar`,`PUT /baselines/{id}`(`BaselineUpdate` 5 字段)
- 删除:`showConfirm` → `DELETE /baselines/{id}` → 回列表
- 同步:`showConfirm` → `POST /baselines/{id}/sync` → 刷新头部 + 同步 Tab

### 6.3 创建向导 `SecBaselineCreatePage`(view `sec-baseline-create`)

- 4 步 Stepper + `FormActionBar`(上一步/取消/下一步/提交):
  1. **基本信息**:`FormField` 网格 — 基线名称* / 编码* / 英文名 / 分类 / 版本(默认时间戳) / 所属产品*(`ProductPicker` 弹窗,搜索 + 树)
  2. **基线来源**:dropzone(点击/拖拽上传 .xlsx)+ 下载模板(`GET /import-template`,blob)+ 上传后调 `POST /preview` 解析;显示解析结果卡
  3. **预览**:统计 tiles(一级/二级/检查项)+ 节点树预览
  4. **确认创建**:摘要 grid + 提示;提交调 `POST /baselines`(FormData:file + 表单字段)→ 跳详情页
- 校验:Step1 名称/编码/产品必填;Step2 必须已解析

### 6.4 组织树管理 `SecBaselineOrgTreePage`(view `sec-baseline-org-tree`)

- `PageHeader`(返回 + "新增根 BG")+ 组织树(可展开/折叠)+ 节点行操作(新增子 / 编辑 / 删除)
- `OrgNodeEditorModal`:name / sort_order / parent(只读)/ node_type(bg/bu/product)
- 删除:`showConfirm` 提示子树节点数 → `DELETE /api/product-org/{id}`
- 数据:`getOrgTree()`,增删改后重取

### 6.5 共用组件

- `OrgTreePanel`:接收 `tree` + `selectedId` + `onSelect`,渲染只读组织树(图标按 node_type:landmark/building-2/git-branch/box),含搜索与"管理组织树"链接
- `ProductPicker`:`Modal` + 搜索 + 产品列表(按 BG/BU 分组)→ 返回 `{id, name}`
- `NodeEditorModal`:`Modal` + `FormField`,按 `node_type` 动态字段(level1:+objective;level2:+sources 字符串数组;item:+priority/is_key_ability/description/verification/sources[{document,section}])
- `OrgNodeEditorModal`:`Modal` + `FormField`(name/sort_order/node_type)

## 7. 数据模型(types.ts)

```ts
export type SyncStatus = 'unsync' | 'syncing' | 'synced' | 'sync_failed';
export type NodeType = 'level1' | 'level2' | 'item';
export type Priority = 'high' | 'medium' | 'low';
export type OrgNodeType = 'bg' | 'bu' | 'product';

export interface BaselineStats { total_items; mapped_items; unmapped_items;
  total_level1_dimensions; total_level2_dimensions; level2_dimensions_with_items;
  level2_dimensions_empty; mapping_coverage_percent: number | string | null; }

export interface BaselineListItem { id; uuid; baseline_name; baseline_name_en?; baseline_code?;
  category?; version?; product_org_id; product_org_name?; total_items?; mapped_items?;
  unmapped_items?; mapping_coverage_percent?; sync_status: SyncStatus; sync_count;
  last_sync_time?; last_updated?; person_id?; person_name?; create_time; }
export interface BaselineWithProduct extends BaselineListItem { bg_name?; bu_name?; }
export interface BaselineDetail { id; uuid; baseline_name; ...; stats?: BaselineStats; }
export interface BaselineUpdate { baseline_name?; baseline_name_en?; baseline_code?; category?; version?; }
export interface BaselinePreview { baseline_name?; ...; stats: BaselineStats; nodes: NodePreviewItem[]; }

export interface NodeOut { id; uuid; baseline_id; parent_id?; node_type: NodeType; code?;
  name; name_en?; objective?; description?; verification?; priority?; is_key_ability?;
  sources: any; sort_order?; person_id?; person_name?; create_time; }
export interface NodeCreate { node_type: NodeType; parent_id?; code?; name; name_en?;
  sort_order?; objective?; sources?; description?; verification?; priority?; is_key_ability?; }
export interface NodeUpdate { /* 同 NodeCreate 但全可选,无 node_type/parent_id */ }
export interface NodePreviewItem { node_type; code?; name; name_en?; parent_code?; ...; sources?; }

export interface OrgNode { id; uuid; parent_id?; node_type: OrgNodeType; name; sort_order?;
  person_id?; person_name?; create_time; }
export interface OrgTreeNode extends OrgNode { children: OrgTreeNode[]; }
export interface OrgNodeCreate { name; sort_order?; parent_id?; node_type: OrgNodeType; }
export interface OrgNodeUpdate { name?; sort_order?; }

export interface SyncResult { success: boolean; message; sync_status: SyncStatus;
  sync_count; last_sync_time?; }
export interface LogOut { id; uuid; target_table; target_id; target_uuid?; action;
  action_detail; person_id?; person_name?; create_time; }
export interface EventOut { id; uuid; baseline_id; baseline_uuid?; target_table; target_id;
  target_uuid?; event_type; from_status?; to_status?; event_detail; person_id?; person_name?; create_time; }
```
