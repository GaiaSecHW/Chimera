# 安全评估项目管理 — 前端实现计划

> 模块路径:`pages/secassessment/sec-assessment-project/`
> 后端仓库:`chimera-ai4eva-platform/sec_func_eval_project_mng_master`(FastAPI :8001)
> 基线服务(跨服务调用):`chimera-ai4eva-platform/sec_func_eval_baseline_mng`(FastAPI :8000)
> UI 原型(参考,非照搬):`referB/安全评估设计/安全评估项目原型` + `安全评估项目原型设计文档.md`
> 分支:`feature/ai4eva_sec-assessment-project`

## 1. 背景与目标

在 Chimera 平台前端实现"安全评估项目"模块,对接已实现的后端 REST API(:8001),并在详情页 Tab3 评估结果中跨服务调用基线服务(:8000)获取基线节点树。

- 开发期前端**直连**两个后端服务(`http://127.0.0.1:8001` 项目管理 + `http://localhost:8000` 基线)
- 预留后期在 **k8s 环境**经 ingress/gateway 访问后端服务的接口(通过环境变量切换,不改代码)
- UI 风格遵循 Chimera 平台设计系统(CSS 变量驱动 + design-system 组件),参考 `referB` 原型但**非照搬**
- 全部改动限定在 `pages/secassessment/` 目录下;一次性例外:`app/navigation.tsx`(嵌套菜单,同基线模块已建立的模式)
- **不改后端任何文件**;不改 `app/viewRegistry.tsx`、`types/types.ts`、`clients/*`、`vite.config.ts`、其他既有模块

### 1.1 关键决策(已确认)

| 决策点 | 选择 | 影响 |
|---|---|---|
| 后端端点缺口 | **前端降级** | 不改后端;详情页 Tab3/4/5 用现有数据实现,缺逐项 execution 处显示降级提示 |
| 实现范围 | **全量** | 4 页 + 详情 8 Tab 全做 |
| 菜单结构 | **嵌套 3 子项** | `app/navigation.tsx` 改 `sec-assessment-project` 为 parent + 3 子项(一次性例外) |

## 2. 调研结论

### 2.1 项目后端(FastAPI :8001)

- **服务**:`sec_func_eval_project_mng_master/app/main.py`,title="安全评估项目 Master 后端",端口 8001
- **路由域**:
  - **项目**(`/api/projects` 前缀):
    - `GET /api/projects?keyword=&status=&sync_status=&offset=&limit=` — 列表(关键字+状态+同步状态过滤+分页),返回 `ProjectListItem[]`
    - `GET /api/projects/{project_id}` — 详情(`ProjectDetail`,含 checkpoint/config_snapshot/chimera_env(key 脱敏)/聚合计数)
    - `PUT /api/projects/{project_id}/executions/{eid}` — 编辑单条评估结果(触发统计重算),返回 `ExecutionResult`
    - `POST /api/projects/{project_id}/sync` — 三方同步(project + 全部 execution),返回 `SyncResult`
    - `POST /api/projects/{project_id}/cancel` — 取消(非终态,保留 checkpoint)
    - `POST /api/projects/{project_id}/re-execute` — 重新执行(仅 cancelled→queued,从头跑)
    - `GET /api/projects/{project_id}/logs` — 操作日志(`OperationLogItem[]`)
    - `GET /api/projects/{project_id}/events` — 事件(`EventItem[]`)
  - **Worker**(`/api/workers` 前缀):
    - `GET /api/workers` — Worker 列表(`WorkerInfo[]`)
    - `POST /api/workers/{worker_name}/drain` — 标记 draining(不抢新任务,等当前完成),返回 `DrainResponse`
  - **配置**(`/api/config` 前缀):
    - `GET /api/config` — 全局系统配置(`SystemConfigRead`)
    - `PUT /api/config` — 更新配置(`SystemConfigUpdate` → `SystemConfigRead`)
  - **M2M Chimera 接口**(`/api/v1/tasks` 前缀):
    - `POST /api/v1/tasks` — 创建任务(`ChimeraTaskRequest` 4 必填 + 扩展 = 建 project + execution + 工作区),返回 `ChimeraTaskCreateResponse`(201)
    - `GET /api/v1/tasks?project_id=&status=&offset=&limit=` — 任务列表(`ChimeraTaskListResponse`)
    - `GET /api/v1/tasks/{tool_task_id}` — 任务状态与进度(`ChimeraTaskStatusDTO`,含 4 阶段 progress)
    - `DELETE /api/v1/tasks/{tool_task_id}` — 删除任务(级联清理 + 幂等)
    - `POST /api/v1/tasks/{tool_task_id}/pause` — 暂停
    - `POST /api/v1/tasks/{tool_task_id}/resume` — 恢复
  - `GET /health`、`GET /api/me`

- **控制操作映射**(8 态状态机):

  | 操作 | 端点 | 走向 |
  |---|---|---|
  | pause | `POST /api/v1/tasks/{chimera_need_taskId}/pause` | M2M |
  | resume | `POST /api/v1/tasks/{chimera_need_taskId}/resume` | M2M |
  | cancel | `POST /api/projects/{id}/cancel` | 项目级 |
  | re-execute | `POST /api/projects/{id}/re-execute`(仅 cancelled) | 项目级 |
  | delete | `DELETE /api/v1/tasks/{chimera_need_taskId}` | M2M(级联) |
  | sync | `POST /api/projects/{id}/sync` | 项目级 |

### 2.2 关键缺口(前端降级处理)

`GET /api/projects/{id}` 返回 `ProjectDetail` 只含**聚合计数**(`total_items` / `finish_count` / `compliance_rate`),**不含逐项 execution 列表**;后端也**无 `GET /api/projects/{id}/executions` 端点**。模型层有 `pass_count` / `partial_count` / `fail_count` / `na_count` / `manual_review_count` 但**不在 `ProjectDetail` 响应 schema 中**。

后端内部已有 `select(BaselineExecution).where(project_id == id)` 查询(`sync_service.py:55`),`ExecutionResult` schema 与 `BaselineExecution` model 均存在,补路由约 10 行即可;但本次按用户决策**不改后端**,采取前端降级:

- **Tab3 评估结果**:展示基线节点树(只读,调 :8000 `GET /api/baselines/{baseline_id}/nodes`)+ 顶部 info 横幅"逐项评估结果需后端补充 `GET /api/projects/{id}/executions` 端点"。`PUT` 编辑端点虽存在,但无法枚举 execution 列表,编辑表单暂不渲染(预留组件,待后端补端点后启用)。
- **Tab4 统计**:用 `ProjectDetail` 聚合计数做合规率仪表盘 + 总项/已完成/未完成进度;分项(PASS/PARTIAL/FAIL/N_A)不在响应中,显示"分项统计需后端补充端点"。
- **Tab5 评估报告**:生成报告(项目元数据 + Chimera 集成 + 聚合统计 + 合规率仪表盘)+ 导出 HTML + 打印;无逐项详情与结论分布图,显示对应提示。

### 2.3 跨服务数据(基线服务 :8000)

- Tab3 左树基线节点树 → `GET /api/baselines/{baseline_id}/nodes`(`NodeOut[]`,前端建树)
- 创建弹窗基线下拉 → `GET /api/baselines`(`BaselineListItem[]`)
- CORS 已含 `localhost:3000`,dev 期直连

### 2.4 数据模型要点

- `project_status`(8 态):`planning` / `queued` / `executing` / `paused` / `cancelled` / `completed` / `failed` / `deleted`
- `execute_status`(4 态):`un_start` / `pending` / `executing` / `finish`
- `execute_result`(5 类):`PASS` / `PARTIAL` / `FAIL` / `N_A` / `MANUAL_REVIEW`
- `confidence`:`high` / `medium` / `low`
- `sync_status`:`unsync` / `syncing` / `synced` / `sync_failed`
- `worker_status`:`online` / `draining` / `offline`
- `agent_engine_type`:`opencode` / `opencode-serve` / `ClaudeCode`
- `max_timeout_unit`:`second` / `minute` / `hour` / `day`
- `config_snapshot`(项目运行配置快照,6+1 字段):`max_retry` / `max_agent_exec_count` / `concurrency` / `max_timeout_value` / `max_timeout_unit` / `agent_engine_type` / `tool_type`;dispatch 时从全局 `sec_system_config` 刷新
- `checkpoint`(4 阶段):`baseline` / `context` / `assessment` / `report`,每阶段 `{status, artifacts[]}`

### 2.5 前端模式(Chimera)

- **HTTP 基础**(`clients/base.ts`):`getHeaders()`(JSON + `Bearer <chimera_token>`)、`handleResponse()`(401 清 token 派发 `chimera-unauthorized`、非 2xx 抽取 detail 拼接并抛带 `.status` 的 Error)
- **设计系统**(`design-system/`):`PageHeader`(含 `back`/`actions`)、`PageSection`、`StatisticCard`、`Toolbar`/`SearchInput`、`DataTable`(列定义含 `render/sortable/align/width`、`onRowClick`、`minWidth`、`showRowNumber`)、`Pagination`、`Modal`、`FormField`、`FormActionBar`、`EmptyState`、`Button`、`Input`、`Select`、`DropdownSelect`、`SegmentedControl`、`Card`
- **反馈**:`showConfirm` / `showAlert` / `showPrompt`(`components/DialogService.tsx`,全局 promise-based)
- **自治子注册表**(`pages/secassessment/viewRegistry.tsx`):框架用 `sec-assessment-` / `sec-baseline-` 双前缀守卫并委托;新增 view 仅在此文件加 case/startsWith 分支
- **嵌套菜单**(基线模块已建立):`app/navigation.tsx` 的 `NavItem` 已支持 `defaultExpanded?` + `subItems?`;`layout/Sidebar.tsx` 已支持 `autoExpanded` 计算含 `defaultExpanded` 分支。本次仅需在 `SIDEBAR_SECTIONS['sec-assessment']` 把 `sec-assessment-project` 从 leaf 改为 parent + 3 子项,**不再改 `layout/Sidebar.tsx`**
- **现有占位页**:`pages/secassessment/sec-assessment-project/SecAssessmentProjectPage.tsx`(开发中占位,本次替换)

## 3. API 接入方案(双服务直连 + k8s 预留)

`client.ts` 顶部:

```ts
import { getHeaders, handleResponse } from '../../../clients/base';

// 项目管理服务(:8001)
const SEC_PROJECT_API_BASE =
  String(import.meta.env.VITE_SEC_PROJECT_API_BASE ?? 'http://127.0.0.1:8001');
// 基线服务(:8000,跨服务取基线节点树)
const SEC_BASELINE_API_BASE =
  String(import.meta.env.VITE_SEC_BASELINE_API_BASE ?? 'http://localhost:8000');
const P_PREFIX = `${SEC_PROJECT_API_BASE}/api`;      // 项目
const P_M2M = `${SEC_PROJECT_API_BASE}/api/v1`;      // M2M
const B_PREFIX = `${SEC_BASELINE_API_BASE}/api`;     // 基线
```

- **Dev**:直连 `http://127.0.0.1:8001` + `http://localhost:8000`,靠后端 CORS;`getHeaders()` 携 chimera_token
- **K8s 预留**:运维设 `VITE_SEC_PROJECT_API_BASE=` / `VITE_SEC_BASELINE_API_BASE=`(空)→ 相对路径,由 ingress 路由
- **网络错误中文化**:`friendlyFetch` 拦截 `failed to fetch` / `networkerror` 等,抛"无法连接项目服务,请检查后端是否启动(默认 127.0.0.1:8001)";基线调用同理
- env 默认值用 `??`(非 `||`),避免空字符串穿透

## 6. 页面设计(参考原型,非照搬)

### 6.1 列表页 `SecAssessmentProjectPage`(view `sec-assessment-project` / `sec-assessment-project-list`)

- 5 `StatisticCard`:排队中(planning+queued)/ 执行中 / 已完成 / 失败 / 平均合规率(已完成项目均值,按当前列表集合计算)
- Toolbar:关键字搜索(项目名/taskId/基线)+ 状态筛选 Select(8 态 + 全部)+ 同步状态筛选 + 结果计数 + 刷新
- `DataTable` 列:
  - 项目名(+ `chimera_need_taskId` 副 mono)
  - 基线(`baseline_name`)
  - 状态徽章(8 态)
  - 负责人(`executor`)
  - 优先级(`priority`)
  - 重试(`retry_count` / `config_snapshot.max_retry`,超限标红)
  - 进度(`finish_count` / `total_items` + 进度条)
  - 合规率(进度条 + %,无值显示 —)
  - 同步徽章
  - 入队时间(`create_time`)
  - Worker(`worker_name`)
  - 操作(按状态启用,见下)
- 行控制按钮(8 态状态机):
  - planning/queued → 删除
  - executing → 暂停 / 取消 / 删除
  - paused → 恢复 / 取消 / 删除
  - cancelled/completed/failed → 重新执行 / 删除
  - deleted → 无
  - 所有非 deleted → 同步到三方(`Share2` 图标)
  - 所有非 deleted → 查看详情(`Eye` 图标,→ `sec-assessment-project-detail-{id}`)
- 行点击 → `sec-assessment-project-detail-{id}`
- 控制操作均带 `showConfirm` 确认 + 状态机警告
- 数据:`listProjects({ keyword, status, sync_status })`;统计卡基于当前列表集合计算

### 6.4 系统配置 `SecAssessmentConfigPage`(view `sec-assessment-config`)

- `PageHeader`(标题"系统配置" + 描述 + 保存/重置按钮)
