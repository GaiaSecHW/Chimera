# WEB端到端用户视图实现计划

## 背景

当前 `task-web-end-to-end` 入口只渲染占位页，位于 `app/viewRegistry.tsx`：

```tsx
case 'task-web-end-to-end':
  return <EmptyPlaceholderPage title="WEB端到端" />;
```

`~/gaiasec` 侧承担开发者视图和运行时能力，包括 agent 注册、worker 调度、MCP 通道、Web 路由采集、任务进度、漏洞上报和漏洞查询。Chimera 这边的 `WEB端到端` 应作为用户视图，不承载底层开发者管理能力。

## 目标

在只能修改 `WEB端到端` 页面相关代码的前提下，实现一个面向项目用户的端到端工作台，覆盖从环境接入到报告查看的核心闭环：

1. 接入环境
2. 查看上线 Agent
3. 展示分析进度
4. 查看漏洞报告

## 非目标

本页面不做以下能力，避免和开发者视图、环境管理视图重叠：

- Agent 部署、重启、卸载、诊断和底层配置管理。
- MCP、worker、pod、进程、数据库、消息队列等运行时细节。
- AI agent/LLM provider 绑定和调参。
- Web route 原始调试台、工具调用日志、原始 session 过程展示。
- 漏洞库的复杂治理流程，例如批量判定、规则管理、研判队列编排。

## 页面信息架构

### 1. 顶部概览

展示 4 个摘要卡片：

- 接入状态：未接入 / 接入中 / 已接入 / 异常。
- 在线 Agent：在线数量 / 总数量。
- 当前分析：无任务 / 执行中 / 失败 / 已完成。
- 报告风险：高危、中危、低危数量。

顶部只回答用户最关心的问题：是否接入、是否有 agent、是否在分析、是否有风险。

### 2. 接入环境

展示内容：

- 当前项目接入状态。
- 接入目标名称或环境名称。
- 最近心跳时间。
- 最小化接入指引。
- 接入异常摘要。

建议交互：

- 未接入时展示接入命令或接入参数说明。
- 已接入时展示接入成功状态和最近活跃时间。
- 异常时只展示可行动的错误摘要，例如 token 无效、agent 离线、无心跳。

不展示底层日志和开发者参数。

### 3. 上线 Agent

展示一个轻量表格或列表：

- Agent 名称或 ID。
- 所属环境。
- 在线状态。
- 最近活跃时间。
- 已识别 Web 应用数量。
- 最近一次分析状态。
- 操作：查看应用、发起分析。

用户视图只暴露可用性和业务含义，不暴露部署/诊断/配置入口。

### 4. 分析进度

用阶段时间线展示当前端到端分析状态：

1. 环境确认
2. Web 应用识别
3. 路由/接口采集
4. 漏洞分析
5. AI 研判
6. 报告生成

每个阶段展示：

- 状态：等待中 / 执行中 / 成功 / 失败。
- 简短说明。
- 更新时间。

失败时展示：

- 失败阶段。
- 失败摘要。
- 重试入口。

不展示 raw task payload、agent tool call、MCP session 和完整日志。

### 5. 漏洞报告

报告区域分为风险摘要和报告列表。

风险摘要：

- 总漏洞数。
- 高危 / 中危 / 低危 / 信息数量。
- 已确认 / 待确认数量。
- 最近报告生成时间。

报告列表字段：

- 漏洞名称。
- 风险等级。
- 影响路径或接口。
- 所属 Web 应用。
- 所属 Agent。
- 状态。
- 更新时间。
- 操作：查看详情、导出报告。

详情可以采用右侧抽屉或页面内详情区，优先展示：

- 漏洞结论。
- 影响范围。
- 证据摘要。
- 复现路径。
- 修复建议。
- AI 研判摘要。

## 数据来源建议

优先复用现有前端 client 能力；如果当前 client 没有对应封装，再在页面局部补充轻量请求函数。

可映射的 gaiasec 后端能力：

- Agent 列表和状态：`/api/agents` 或项目维度 agent 查询接口。
- Agent 任务：`/api/tasks/agents/{agent_id}`。
- Web 应用/路由：`/api/agents/{id}/web-routes` 或同类 route 查询接口。
- 漏洞列表：`/api/vulnerabilities/projects/{project_id}`。
- 漏洞详情：`/api/vulnerabilities/{id}`。

如果 Chimera 当前 API 代理路径与 gaiasec 不完全一致，页面实现时以现有 `clients/*` 的 API base 和响应处理方式为准。

## 实现范围

建议新增页面：

- `pages/task/WebEndToEndPage.tsx`

修改路由注册：

- `app/viewRegistry.tsx`

只将 `task-web-end-to-end` 从占位页替换为 `WebEndToEndPage`。

除非现有 client 已经明确需要复用，否则不要改动其他任务页、环境页、漏洞治理页和导航结构。

## 组件拆分建议

页面内部可拆成局部组件，先不抽公共组件：

- `SummaryMetricCard`
- `AccessEnvironmentPanel`
- `OnlineAgentPanel`
- `AnalysisProgressPanel`
- `VulnerabilityReportPanel`
- `VulnerabilityDetailDrawer`

这些组件先放在 `WebEndToEndPage.tsx` 内部，避免过早公共化。

## 状态设计

页面状态建议保持简单：

- `loading`
- `error`
- `agents`
- `selectedAgentId`
- `currentTask`
- `progressStages`
- `vulnerabilities`
- `selectedVulnerability`

刷新策略：

- 首次进入页面加载。
- 提供手动刷新按钮。
- 分析进行中时可 10 到 15 秒轮询一次。
- 页面卸载时清理轮询。

## 交互流程

### 未接入

1. 顶部显示接入状态为未接入。
2. 接入环境区域展示接入说明。
3. Agent、分析进度、报告区域展示空态。

### 已接入但无 Agent

1. 顶部显示接入中或异常。
2. Agent 区域展示无在线 Agent。
3. 引导用户确认接入环境。

### 有在线 Agent

1. Agent 区域展示在线 Agent。
2. 用户选择 Agent。
3. 页面展示该 Agent 已识别的 Web 应用和最近分析状态。
4. 用户点击发起分析。

### 分析中

1. 分析进度区域展示阶段时间线。
2. 顶部当前分析卡片显示执行中。
3. 报告区域展示已有历史报告或等待报告生成。

### 分析完成

1. 顶部当前分析卡片显示已完成。
2. 报告区域刷新漏洞列表和风险摘要。
3. 用户可查看详情或导出报告。

## 空态和错误态

必须覆盖：

- 未选择项目。
- 接口加载失败。
- 无接入环境。
- 无在线 Agent。
- 无分析任务。
- 无漏洞报告。
- 分析失败。

错误文案应该给用户可执行动作，例如“刷新状态”“检查 Agent 是否在线”“重新发起分析”。

## UI 风格

遵循当前 Chimera 页面风格：

- 使用 Tailwind class。
- 卡片圆角不超过现有页面习惯。
- 信息密度适中，偏运维/安全工作台，不做营销式 hero。
- 操作按钮使用清晰动词，例如“刷新”“发起分析”“查看详情”“导出报告”。
- 风险等级用稳定颜色：高危红色、中危橙色、低危蓝色、信息灰色。

## 分阶段落地

### 阶段一：静态页面骨架

- 新增 `WebEndToEndPage.tsx`。
- 实现顶部摘要、四个主体区域和空态。
- 将 `task-web-end-to-end` 接入该页面。
- 使用 mock/派生空数据，确保页面可见且结构稳定。

### 阶段二：接入真实 Agent 和任务数据

- 拉取当前项目 Agent。
- 派生接入状态和在线 Agent 数量。
- 选择 Agent 后拉取最近任务。
- 用任务状态生成分析进度时间线。

### 阶段三：接入漏洞报告

- 按项目拉取漏洞列表。
- 支持按 Agent 或 Web 应用过滤。
- 生成风险摘要。
- 实现漏洞详情抽屉或详情区。

### 阶段四：分析触发和报告导出

- 为在线 Agent 提供发起分析入口。
- 分析中开启轻量轮询。
- 支持导出当前报告或打开报告详情。

## 验证计划

本地验证：

- `npm run build`
- 页面可从侧边栏进入 `WEB端到端`。
- 未选择项目时展示明确空态。
- 无数据时页面不报错。
- mock 或真实接口返回部分字段缺失时页面仍能渲染。

交互验证：

- 刷新按钮不会重复并发请求。
- 分析中轮询能清理。
- Agent 切换后报告和进度随选择更新。
- 漏洞详情打开和关闭正常。

## 风险和处理

- 后端 API 路径可能与 gaiasec 原生接口不同：实现时优先查看 `clients/*` 已有封装，必要时只在页面内封装最小请求。
- Web route 和 task 的状态模型可能不统一：页面以用户可理解阶段做映射，不直接暴露原始状态。
- 漏洞报告数据可能只有漏洞列表、没有正式 Markdown 报告：先用列表和详情承载报告能力，导出可后置。
- 当前页面只能改 `WEB端到端`：不要改动环境管理、漏洞治理和开发者页来补能力。
